The Memory of a Potion

Game programmer & Game and Level Designer

Media Gallery

Project Overview

Created a story puzzle game with clues around the environment. The game features puzzles that utilize environmental interactions and mechanics to challenge players. The design focused on creating puzzles that are intuitive, engaging and rewarding. This project was a solo project where I was responsible for all aspects of puzzle design, including concept development, prototyping, playtesting, and iteration. The goal was to create a cohesive puzzle experience that encourages exploration and using your surroundings to solve puzzles and remember your story.

Duration

1 month

Team Size

1 person (Solo Project)

Roles

Game Programmer

Level Designer

Game Designer

Tools

Unity Engine

Key Responsibilities

  • Designed the environmental interaction mechanics and systems
  • Created puzzles with clear visual and mechanical feedback
  • Prototyped puzzle concepts and iterated based on player feedback
  • Programmed puzzle mechanics and systems in Unity
  • Implemented a hint system that provides contextual clues without giving away solutions

Design Highlights

Elegant Mechanics

Core mechanics are simple to learn but allow for complex puzzle designs and fun solutions.

Story Integration

Puzzles are designed to reinforce the game's narrative, ensuring that solving puzzles feels meaningful and contributes to the overall story experience.

Hint System

Implemented a contextual hint system that provides players with helpful clues without revealing puzzle solutions.

Visual Clarity

Environmental design clearly communicates which objects are interactive and what interactions are possible.

Playtesting Insights

  • Clear visual feedback is essential for players to understand puzzle mechanics
  • Hint systems should be integrated naturally into the game world
  • Players prefer puzzles in their intended environment rather than abstracted or simplified versions
  • Making a puzzle feel like it belongs in the game world is key to player engagement
  • Programming potion making in a way that is easy to expand and maintain is key to future development
